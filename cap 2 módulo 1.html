Representação de dados, o que é bit e bytes?

O computador é um equipamento eletrônico, e como todo equipamento eletrônico ele funciona com sinais e esses sinais pra computação são representados como 0 e 1 que são chamados de Binary Digit ou Simplificando: 'Bit', mas um 0 e um 1 não dá pra representar muita coisa por isso a computação ela reúne um conjunto de 8 bits que é a porção miníma pra que se represente algum dado, essa sequência de 8 bits é chamada de Byte, então 1 Byte é uma sequência de 8 bits, 1 byte é a porção miníma que pode representar uma determinada informação. Exe: 01000001 = A, essa sequência de bits representa a letra A, Quando vc aperta a letra A no seu teclado o que vai pra memória do seu computador é essa sequência de bites. Obs. Se você quiser saber qual a tabela inteira de representação em bytes pesquise por: Código Multibyte UTF-8

Os múltiplos do byte:

O computador só entende essas ondas binárias, ou ondas quadradas. Assim como o byte a gente tambêm tem os multiplos do byte, Quando vc pega sequências de bites vc tem multiplos sempre representados em 1024 bytes que é igual a 1KB, quando eu tenho 8 bits eu tenho 1 byte, quando eu tenho 1024 bytes eu tenho 1KB e os números vão só crescendo então basicamente: 1024bytes = 1KB, 1024KB = 1MB, 1024MB = 1GB, 1024GB = 1TB, 1024TB = 1PB, 1024PB = 1EB, 1024EB = 1ZB, 1024ZB = 1YB.

Quando vc pega 1MB você está se referindo a 1024 Megabytes, e quando vc pega 1Mb você está se referindo a 1024 Megabites, o B maíusculo é bytes e o mínusculo bites. Normalmente na área de tecnologia a gente usa o B maíusculo pra se referir a armazenamento Exe: ssd, pen drive.. e o B mínusculo para se referir a transmissão Exe: rede, placa de rede..

Oque é um modem? Como nos conectamos a internet?

O seu computador, celular, tablet etc.. é chamado de cliente, e atráves de um modem que vai fazer a transformação é que vai ser possivel vc ter acesso a internet. O modem é um aparelho que vai fazer o processo de transformação de uma onda para a outra e vice-versa, que é chamado de modulação e demodulação.

Como acessamos os servidores?

Suponha que vc quer acessar o 'instagram', vc vai lá e pesquisa instagram.com, e a internet vai buscar dentre um servidor ou conjunto de servidores pelas informações que vc procura. Mas ele não vai ser identificado por um nome assim como vc tá vendo ai, pq toda máquina que tá na internet é representada por um número, e até vc quando se conecta a internet recebe um número desses, que é chamado de Endereço IP. Obs. Quando vc liga e desliga seu modem o seu endereço IP já muda para outro.

Exe: quando vc pesquisa por instagram a internet encontra o número do servidor, mas como ela sabe que aquele IP é referente ao instagram? então.. a internet tem algo como uma 'agenda digital' chamada de Servidor 'DNS' essa sigla sgnifica 'Domain Name System' ou sistema de nome de dominío. Ou seja, quando vc pesquisa por instagram a informação não vai direto pro servidor e sim pro sistema de 'DNS' depois pra sua máquina, e por último pro servidor que vai te mandar a informação desejada.

O que são rotas?

A internet funciona baseada em rotas, ela é uma rede finita com caminhos finitos por exemplo: quando vc solicita um dado ou acessa um site ela vai procurar a melhor rota pra vc. Perçeba que as vezes vc acessa um site e demora bastante dai quando vc aperta f5 ou recarregar ele consegue acessar mais rápido, isso quer dizer que vc pegou uma rota diferente da anterior, uma mais rápida.

Domínio e Hospedagem:

Os dados que estão em um servidor podem ser acessados por qualquer pessoa desde que ela pesquise pelo seu endereço de domínio, já se os dados estiverem em seu computador pessoal eles só poderão ser acessados por você, pois não há um endereço que o dns possa encontrar e ter acesso.

Basicamente a URL é a ponte que vai dar acesso ao DNS que vai te dar acesso ao servidor em específico, e pra ter esse acesso a primeira coisa que vc precisa ter é um DOMÍNIO, pra que vc possa acessar e dar acesso a outras pessoas do 'site' que vc criou. Na vdd vc precisa de um domínio e tambêm de um Servidor. Um servidor é basicamente um local onde vc vai hospedar seus arquivos, para que vc e outras pessoas tenham acesso a eles.

Domínio:

Domínio é o nome que identifica o seu site. O Domínio é um nome único, não podem existir dois domínios com o mesmo nome, quando vc vai colocar um domínio ele vai dizer se aquele nome já existe ou não se um domínio já existir vc não vai conseguir usar. Tambem quando vc criar um domínio vc precisa pagar um plano anual para poder usá-lo. E na hora de escolher vc vai ter que escolher um dos vários TLDs, que significam 'Top Level Domains'.

Hospedagem:

Hospedagem: É o local onde o seu site vai estar amrmazenado. O espaço para armazenar seus arquivos, é como se fosse um outro computador que vc pode armazenar coisas lá dentro, Essa hospedagem normalmente é paga mensalmente. Existem vários tipos de hospedagens, vc tem que escolher uma de acordo com Seus requisitos de: Espaço, Memória, Recursos.

O que é uma URL?

Url é uma sigla para 'Uniforme Resource Locator' que siginifica Localizador de recurso único ou uniforme. É um endereço que só serve pra te desviar para um lugar em especifíco. dentro de uma Url a gente tem várias partes.

Exe: www.github.com/gustavogauanabara O 'github.com' é um DOMÍNIO dentro desse Domínio a gente tem o '.com' que é uma 'TLD', ou seja o final do domínio é sempre uma TLD ou GTLD, esse G significa Genérico.

Podendo ser: '.com' (geralmente comercial) '.edu' (instituiçôes educacionais) '.gov' (instituições governamentais) '.store' (geralmente para lojas) esses são os domínios GTLDs. E existem tambem os CCTLDs esse 'CC' significa Country Code que é código de pais.

Exe: '.br'(para o brasil) '.us' (para os EUA) '.uk' (para a inglaterra) '.ar' (para a argêntina) e etc.. o 'www' É chamado de Sub-Domínio, no caso ele é o sub-domínio principal do seu servidor web. Já essa '/gustavoguanabara' É chamado de caminho, então Basicamente uma Url é composta por um: Sub-Domínio, Domínio, Tld e Caminho.

Um outro Exemplo de uma Url: gustavoguanabara.github.io O 'gustavoguanabara' é o Sub-Domínio, o 'github.io' é o Domínio e o '.io' a nossa Gtld. Obs. O https é o prótocolo, esse 'S' significa segurança, ele utiliza ssl.

Página desenvolvida para adicionar dicas de códigos e funções em HTML5 e CSS3.

Semântica na HTML5:

Semântica: Ciência que estuda a evolução do significado das palavras, signos e símbolos, que estão a serviço da comunicação. (O significado dos vocábulos, por oposição a sua forma). Analisando a frase acima percebemos que uma palavra pode ter forma e significado, e que a Semântica dá mais valor ao significado. A maior mudança da HTML4 para a HTML5 é o acréscimo das Semânticas aos elementos, a Semântica em HTML é 'significado', o HTML5 não se foca mais em forma, se foca em semântica (em significado), já o CSS3 se foca mais em Estilos e Formas.

Na HTML4, tínhamos tags como <b> que colocava um texto em negríto, <u> que colocava o termo sublinhado e <blink> que fazia o texto piscar. Essas eram tags que representavam apenas uma forma. Você dizia que queria um texto sublinhado, mas qual era o motivo? Qual era o sentido de sublinhar alguma coisa? Vamos ver um exemplo:

<p> Juvenal era um sujeito de muita sorte. E já começou de pequeno, onde morou na <u> Rua Marquês de Lira Filho</u>, um local de fácil acesso ao centro da cidade.</p>

Neste exemplo acima usamos o par de tags <u> e </u> para sublinhar o termo 'Rua Marquês de Lira Filho' e chamar a atenção do leitor ao endereço onde o cara nasceu. Esse sublinhado em HTML4 é apenas uma forma visual para chamar a atenção, sem um significado explícito. Já a HTML5 chegou com o conceito de valorizar a semântica, logo suas tags tentam levar um significado muito forte embutido. Então a frase acima ficaria:

<p> Juvenal era um sujeito de muita sorte. E já começou de pequeno, onde morou na <address> Rua Marquês de Lira Filho</address>, um local de fácil acesso ao centro da cidade.</p>

Note que agora usamos a tag <address> e </address> para dar um significado ao sublinhado em destaque que fizemos antes (o endereço da pessoa). Nesse caso em um navegador no seu celular; por se tratar de um endereço ele poderia até sugerir que você veja o mapa do local e trace uma rota para chegar lá. Viu? Significado! Sendo assim, em HTML5, vemos de forma bastante evidente a presença do chamado HTML Semântico ou Tags Semânticas ou ainda o Conteúdo Semântico. Você quer que um endereço apareça na forma de um texto sublinhado? Então use CSS para configurar isso, a HTML serve apenas para dar sentido ao conteúdo, A CSS para dar Significado, é assim que tudo vai funcionar.

Desde a mudança das versões da HTML, a W3C (consórcio responsável por normatizar a HTML) tem dado muito valor por adicionar novas tags com significados e remover algumas que só se focam em efeito visual (formas de apresentação). O intuito é deixar a apresentação gráfica por conta das CSS.

Existem tags que ainda funcionam hoje em dia, mas estão prestes a serem consideradas obsoletas. Isso acontece porque a linguagem evolui, e nesse processo algumas param de fazer sentido pois existem outras bem melhores para usar e que fazem sentido. Por isso sempre fique atento as mudanças de tags, abaixo vou deixar alguns sites que são atualizados constantemente com as tags que estão ficando Obsoletas e algumas substituições desejáveis que devemos fazer.

Formatação de Textos, Negrito e Itálico:

Algumas das tags mais usadas nas versões anteriores do HTML são <b> Negrito e <i> itálico, mas elas não possuem significado e focam apenas na forma, sendo assim são pouco semânticas. Então ao invês de usarmos elas utilizamos as tags <strong> e <em>, a tag <strong> significa que o termo dentro dela possui força dentro da frase. Logo ele aparecerá em negrito. E a tag <em> (ênfase) Significa que queremos dar Ênfase ao termo. Logo ele aparecerá em itálico.

Note que, ao usar <strong> e <em> no lugar de <b> e <i>, damos mais significado aos nossos termos e conteúdos. Como eles vão ser representados visualmente (forma) vai depender das nossas folhas de estilo CSS.

Então: Tag <b> para termo em negrito (sem semântica). Tag <i> para texto em itálico (sem semântica). Tag <strong> para termo em negrito (com semântica). Tag <em> para termo em itálico (com semântica). E também temos a Tag <mark> para demarcar um termo como se estivessemos usando uma caneta marcador (marca-texto).

Teclas de atalho no VSCode:

Ctrl+Shift+P: Vai abrir um menu de opções no VSCode, após isso selecione a Opção 'Wrap with abbreviation' depois escolha a tag que vc quer utilizar e tecle "Enter" duas vezes. Obs. antes disso selecione a mensagem que vc quer, depois Ctrl+Shift+P + Exe:(tag <i>), Enter + Enter.

Formatações adicionais em HTML:

Texto Demarcado:

Além das tags STRONG e EM também podemos utilizar a tag MARK para demarcar um texto, e também podemos usar outro método agora em CSS para mudar a cor do marcador.

Exe: <p>Podemos criar também <mark style="background-color: blue;">um texto marcado</mark> usando a tag MARK.</p>

Mas esse método só irá funcionar nesta única tag mark, então vamos supôr que vc queira criar e demarcar outras tags 'Mark', para isso temos um outro método abaixo. Na área <head> logo abaixo do <title> vc irá escrever:

                <style>
                    mark {
                    background-color: blue;
                    }
                </style>

Texto Grande / Texto Pequeno:

Agora vamos supôr que vc queira criar um texto GRANDE ou PEQUENO, antigamente para fazer isso utilizavamos as tags <big> (para grande) e <small> (para pequeno) mas a tag <big> se tornou obsoleta pois ela não não tinha forma, servia apenas para aumentar um pouco o texto, e para isso nós podemos utlizar o CSS. Mas a tag <small> ainda funciona e é utilizavel mesmo não sendo semântica.

Então o <big> caiu pois não tinha sentido, se vc queria uma letra maior basta apenas formata-la em CSS. Já o <small> continua funcional, pois por exemplo em um "contrato" que tem aquelas letras pequenas; para isso foi utilizado o <small> então ele acaba tendo um sentido.

Texto Excluído / Texto Inserido:

Agora vamos supôr que vc quer criar um parágrafo com um texto que foi EXCLUÍDO ou SUBLINHADO (com semântica), para isso vc utilizaria as tags: <del> (para excluído) e <ins> (para adicionado). Nesse caso a tag <del> seria para um termo que deve ser lido mas não considerado. E a tag <ins> para dar uma ênfase a um termo que foi adicionado depois.

Texto Sobrescrito / Texto Subscrito:

Agora quantas vezes você tentou escrever um x² ou um h2o em um documento e ficou na dúvida de como fazer para esses números ficarem pequenininhos ali em cima ou embaixo. Pois bem, para isso a HTML possui as tags <sub> (para deixar sobrescrito) e <sup> (para deixar subscrito) justamente para essas finalidades.

Citacões e Códigos:

Outras Formatações:

Obs: na criação do arquivo "index.html" o index significa indíce, ou seja a Pagina principal, e como o ex09 só tem uma página ela é a principal.

Tag <code> para fonte mono-espaçada, geralmente usada para ler códigos.

Tag <pre> junto com a tag <code> é usada para criar um código com índentação. Obs. tudo que estiver dentro do <pre> vai ser considerado.

Dica: Shift+Tab para remover uma indentação (aqueles espaços antes do código), e Tab para criar uma índentação. Isso só irá funcionar selecionando apenas o código, sem as Tags.

Citações Simples:

Tag <q> para criar uma citação simples, sem quebra de linhas (semãntica).

Citações Completas:

Tag <blockquote> para criar uma citação completa com quebras de linhas e indexação do url de origem. Uma blockquote geralmente aparece deslocada um pouco pra dentro, para mostrar que ela é uma citação a um autor.

Parâmetro 'Cite' que significa Citando é usando dentro de uma blockquote para anexar a (url) de origem daquele documento a frase citada. O cite é usado para anexar a url daquela citação, mantendo os direitos autorais do seu autor.

Exe: <blockquote cite="url" + frase </blockquote>

Abreviações:

Tag <abbr title=""> usada para criar e mostrar o significado daquela abreviação ao passar o cursor do mouse encima dela. Exe: Abreviação de HTML, Seria; <abbr title="HyperText Markup Language">

Texto Invertido:

Tags <bdo dir="ltr"> (para criar uma citação de trás pra frente) ou <bdo dir="rtl"> (para criar uma citação normal).

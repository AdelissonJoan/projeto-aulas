Diferença entre HTML, CSS e JAVASCRIPT:

Como funciona a html e as Css?

Tanto a HTML quanto as CSS são linguagens, mas não linguagens de programação. Então quando vc for falar que trabalha com elas vc deve dizer: Eu desenvolvo em HTML e CSS.

"HTML" siginifica: 'HiperText Markup Language' que é Linguagem de Marcação para HiperTexto, HiperTexto é quando vc tem uma área em que pode clicar e ela vai te direcionar para outro texto, ou imagem etc.. a linguagem Html é uma linguagem fundamentada em marcas, por isso ela não é uma linguagem de programação e sim de marcação. Basicamente a HTML ela é uma linguagem focada em conteúdo, podendo ser ele: textos,imagens, videos, tabelas, listas e etc.. tudo que for focado em conteúdo é HTML.

"CSS" significa: 'Cascading Style Sheets' que são Folhas de Estilo em Cascata, Cascata por que uma vai modificando a caracteristica padrão da outra, os seletores vão mudando e acrescentando umas novas caracteristicas em cascata. já a CSS é basicamente focada em design, sendo eles: cores, sombras, tamanhos, posicionamentos e etc..

"JAVASCRIPT" o javascript é uma linguagem de PROGRAMAÇÃO, em um site ele é basicamente as interações que você pode ter, sendo elas em: menus, animações, popups, validações e etc..

Extensão para baixar que mostra a diferenças de cada uma dessas linguagem em um site: Google crhome, Web Developer.

Então basicamente HTML = Conteúdo, CSS = Estilo e JAVASCRIPT = Interatividade.

Como criar um conteúdo em HTML?

Vamos supôr que vc queira criar um titúlo pro seu site, então seria: <h1> Exemplo de Titúlo <h1> , esse </h1> seria a abertura de tag, esse "Exemplo de Titúlo" seria o conteúdo e esse </h1> o fechamento de tag.

Agora vamos supôr que vc queira criar um parágrafo, então seria: <p> Exemplo de Parágrafo </p>

Agora vamos supôr que vc queira adicionar uma imagem, então seria: <img.src="foto.png" alt="Exemplo de Foto"> , esse "<img" é a nossa abertura de tag (ele é uma das Tag's que não possuem fechamento) esse 'src e alt' são parâmetros, e o 'foto.png e Exemplo de Foto' são Valores.

SRC vem de Source que é origem, então eu tô colocando uma imagem cuja a origem é o arquivo 'foto.png' e o texto alternativo (alt) dele é 'Exemplo de Foto' , então se eu não conseguir mostrar a imagem de qualquer maneira ele vai me mostrar o Exemplo de Foto. Toda Tag pode ter parâmetros e valores eu posso ter tambem parâmetros e valores em exemplos de titúlos e exemplos de parágrafos, mas é menos comum.

Como adicionar estilos em CSS?

Agora vamos supôr que vc quer pegar esse <h1> que é um titúlo, e mudar a sua cor, tamanho ou seu tipo de fonte, então seria:

                h1 { font-family: Arial;
                        font-size: 20pt;
                        color: Blue;
                    }

Esse 'h1' seria o Seletor, o 'font-family:' seria a Declaração, ou seja: estaria declarando que agora a sua fonte seria a Arial. Esse 'font-size:' significa tamanho da fonte, ou seja: o tamanho da fonte vai ser 20 pontos, tambem podendo ser 'px' que significa Pixels. Esse 'color:' significa cor da fonte, ou seja: a cor da fonte vai ser azul. 'color' é o que a gente chama de propriedade, e 'blue' é o que a gente chama de Valor.

Obs. Um estilo em CSS ele está geralmente dentro de uma Tag 'Style' em HTML, mas ele utiliza Seletores, e Toda declaração em CSS tem ';' no final. Então Basicamente CSS tem: Seletores, dentro dos seletores a gente tem declarações e cada declaração é um par de propriedade e valor.

Estrutura básica de um documento HTML:

A primeira linha em HTML tem que dizer que tipo de documento vc vai desenvolver, então : <!DOCKTYPE html> Depois eu vou começar a montar a estrtura em HTML e vou dizer em qual linguagem essa estrtura será feita: <html lang="pt-br">

Todo HTML tem duas áreas bem definidas, a área da cabeça que é '<head>' (a área head é a área de configurações) e a área do corpo que é '<body>', o corpo é aquela área onde fica o conteúdo do seu site, as matérias, os menus etc..

Essa linha '<meta charset="UTF-8">' Significa que o meu site vai ser compatível com caracteristicas de "UTF-8" como por exemplos as acentuações.

Essa outra linha '<meta name="viewport" content="width=device-width, initial-scale=1.0">' Significa que quando o seu site vai utilizar toda a parte branca em uma escala de 100% e quando vc abri-lo ele vai estar na escala normal e não em zoom.

Essa linha '<title>Document</title>' Significa o titúlo que vai aparecer lá no começo do seu site.

Como vai ser a Estrutura:

                <!DOCKTYPE html>
                <hmtl lang="pt-br">
                    <head>
                        <meta charset="UTF-8">
                        <meta name="viewport"
                        content="width=device-width,
                        initial-scale=1.0">
                        <title>Document</title>
                    </head>
                    <body>
                        <h1>Olá, Mundo!<h1>
                    </body>
                </html>

Como funcionam a HTML e as CSS?

Assim como vimos anteriormente: A sua máquina vai acessar a url, essa url vai ser direcionada ao DNS que vai te retornar um endereço IP que vai te levar a um servidor onde vc vai poder acessar esse conteúdo. O servidor vai te retorna uma CÓPIA desse conteúdo e ai o seu navegador vai ser o responsável por analizar cada linha daquele código em HTML e CSS pra poder gerar e te enviar um conteúdo visual.

HTML é apenas o esqueleto do site, CSS é o design do site, javascript é a interação que o site vai fornecer para o usuário em questões de click. Isso tudo que eu mencionei é a parte do front end! Back end é o que o usuário não consegue ver! O back end fica no servidor, lá está o banco de dados, e as linguagens de programação que manipulam o banco de dados são: Python, PHP, Java etc..

Diferença entre Front-End Back-End e Full-Stack:

Front-End ou Client-Side:

O Front-End é aquele cara responsável por criar aquela parte visual e interativa que o cliente vai ter com o site, ele sempre vai se preocupar muito em gerar uma boa experiência visual do cliente com o site.

Já a função de um servidor é armazenar e enviar dados para o cliente (no caso a sua máquina), normalmente nos casos mais simples o servidor vai te enviar 3 tipos de tecnologias que são o: HTLM, CSS e o JAVASCRIPT e assim como ele envia pra vc ele vai enviar pra todos os outros clientes que acessarem aquele site e solicitarem a mesma página, isso é o que a gente chama de site estático, onde o mesmo site é entregue para todos os clientes.

Exe: Se eu e você estiver acessando agora o mesmo site (estático) a nossa experiência visual e interativa vai ser exatamente a mesma. Então o Front-End tá focado mais no usuário, na experiência dele, e na interatividade dele com o site.

Todas essas tecnologias citadas anteriormente o HTML, CSS e JAVASCRIPT são tecnologias que a gente diz que vão funcionar do lado do cliente, e o seu navegador seja ele qual for ele é capaz de executar essas 3 linguagens, e esse tipo de tecnologia é chamado de CLIENT-SIDE ou FRONT-END. São tecnologias que vão funcionar do lado do cliente e não no servidor o papel do servidor em tecnologias Client-Size é simplesmente entregar o arquivo pra você, e o seu navegador vai ser o responsável por executar tudo isso.

Então basicamente o desenvolvedor Front-End Web é alguém que vai desenvolver coisas em HTML, CSS e JAVASCRIPT, tecnologias que vão rodar do lado do cliente. Mas é claro que o desenvolvedor Front-End vai precisar ter alguns outros conhecimentos desejáveis, como por exemplo: saber o basíco de Imagens, de Cores, de Tipografias, de Photoshop de Gimp.. Mas tudo isso não significa que o desenvolvedor Front-End vai desenvolver o Design, Então não confunda desenvolvedor Front-End com Designer.

Dica de Cursos com os fundamentos da programação: Visualg e o Portugol studio.

Back-End ou Server-Side:

Por outro lado existem tecnologias que vão rodar dentro do servidor como por exemplo o: Php, Python, JS Noodle, C#, Ruby, Java.. Essas tecnologias são chamadas de Tecnologias Server-Side ou Back-End. Então o Back-End é aquele cara responsável por armazenar, selecionar, analisar e enviar dados do servidor para o cliente. E o Back-End tá focado mais na interação do código dele com o servidor, nas entranhas do servidor, com as tecnologias envolvidas na tecnologia de Server-Side.

Agora vamos supôr um exemplo: vamos imaginar que o cliente está tentando acessar uma loja online para comprar uma camiseta, se por acaso o seu cliente fizer uma solicitação para o servidor e o servidor tiver um site estático em HTML CSS e JS puros, ele vai te retornar com os arquivos e o cliente será o responsável por montar aquela estrutura ali na tela.

Pórem esse site não pode ser estático pois toda vez que o cliente fosse acessar o estoque ou escolher o tamanho da camiseta; o dono daquela 'loja online' teria que mexer no código para mudar alguma coisa. Então é ai que entra o papel do Back-End, hoje em dia todas as informações de uma 'loja online' como por exemplo o: estoque, cores disponiveis, seus tamanhos.. essas informações estão todas armazenadas em um banco de dados dentro do servidor.

Sendo assim o mecanismo se torna um pouco diferente de um site estático, basicamente quando há o envolvimento de tecnologias Server-Side o que acontece é o seguinte: O cliente vai enviar uma solicitação para o servidor, no servidor a linguagem (PHP ou etc..) vai fazer uma solicitação para esse banco de dados, o (PHP ou etc..) vai obter essas informações solicitadas e vai gerar uma página em HTML puro e depois vai enviar essa página em HTML junto com (JS, CSS) para o cliente. A partir daí com essas informações e essa página em HTML feita em (PHP ou etc..) o cliente vai ter todas as informações daquele banco de dados sobre todos os produtos naquele site.

Full-Stack:

Então se o desenvolvedor quiser ser um Front-End ele tem que ser especializado em tecnologias como: HTML, CSS e JS, e quanto mais ele conhecer essas tecnologias melhor ele vai ser em Front-End pórem se ele prentende conhecer mais sobre Banco de Dados, Acesso a Dados, Captura de Informações via Servidor .. ele deve se tornar tambêm um desenvolvedor Back-End e se especializar nessas tecnologias. Então é ai que entra o Desenvolvedor Full-Stack, ou seja: Alguem com os conhecimentos capaz de desenvolver tanto em Front-End quanto em Back-End, o Full-Stack é a junção desses dois conhecimentos.
